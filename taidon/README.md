# Taidon - SQL Editor & Project Management System

![Taidon Logo](https://via.placeholder.com/200x80/007bff/ffffff?text=Taidon)

Taidon - это современная веб-платформа для создания, редактирования и управления SQL скриптами с поддержкой совместной работы и управления проектами.

## 🚀 Основные возможности

### 📝 SQL Редактор
- **Синтаксическая подсветка** для SQL с поддержкой различных диалектов
- **Автодополнение** и интеллектуальные подсказки
- **Анализ синтаксиса** в реальном времени
- **Множественные темы** оформления (светлая, темная, и другие)
- **Поиск и замена** с поддержкой регулярных выражений

### 🗂️ Управление проектами
- **Создание и организация** SQL проектов
- **Иерархическая структура** файлов и скриптов
- **Копирование проектов** с возможностью выбора содержимого
- **Публичные и приватные** проекты

### 👥 Совместная работа
- **Система пользователей** с аутентификацией
- **Права доступа** к проектам (чтение, запись, администрирование)
- **Совместное редактирование** скриптов
- **История изменений** и версионирование

### 💾 Автосохранение
- **Автоматическое сохранение** изменений
- **Синхронизация** с сервером в реальном времени
- **Работа в офлайн режиме** с последующей синхронизацией

## 🏗️ Архитектура

### Frontend
- **React 18** - современная библиотека для создания пользовательских интерфейсов
- **CodeMirror 6** - мощный редактор кода с поддержкой SQL
- **Webpack 5** - сборщик модулей и оптимизатор
- **CSS Modules** - изолированные стили компонентов

### Backend API
- **RESTful API** с полной документацией OpenAPI 3.0
- **JWT аутентификация** для безопасности
- **Система прав доступа** с гранулярным контролем
- **Автоматическая валидация** входящих данных

### Интеграция
- **Полная интеграция** фронтенда с бэкендом
- **Обработка ошибок** и graceful degradation
- **Кэширование** для улучшения производительности
- **Mock режим** для разработки без бэкенда

## 🔧 Конфигурация

### Настройка API
Для подключения к бэкенду отредактируйте файл [`src/api/backendClient.js`](src/api/backendClient.js):

```javascript
const API_BASE_URL = 'http://localhost:8000'; // URL вашего бэкенда
const USE_MOCK_MODE = false; // Отключить mock режим для работы с реальным API
```

### Настройка Webpack
Конфигурация сборки находится в файле [`webpack.config.js`](webpack.config.js). Основные настройки:

- **Точка входа**: `src/index.js`
- **Выходная папка**: `dist/`
- **Dev Server**: порт 3000 с hot reload
- **Babel**: транспиляция JSX и современного JavaScript

### Переменные окружения
Создайте файл `.env` в корне проекта:

```env
REACT_APP_API_URL=http://localhost:8000
REACT_APP_ENVIRONMENT=development
REACT_APP_VERSION=1.0.0
```

## 📖 API Документация

Полная документация API доступна в файле [`openapi.yaml`](openapi.yaml). Основные эндпоинты:

### Аутентификация
- `POST /auth/register` - Регистрация пользователя
- `POST /auth/login` - Вход в систему
- `GET /auth/me` - Получение информации о пользователе
- `PUT /auth/me` - Обновление профиля

### Проекты
- `GET /projects/` - Список проектов
- `POST /projects/` - Создание проекта
- `GET /projects/{id}` - Получение проекта
- `PUT /projects/{id}` - Обновление проекта
- `DELETE /projects/{id}` - Удаление проекта
- `POST /projects/{id}/copy` - Копирование проекта

### Скрипты
- `GET /scripts/project/{project_id}` - Скрипты проекта
- `POST /scripts/` - Создание скрипта
- `GET /scripts/{id}` - Получение скрипта
- `PUT /scripts/{id}` - Обновление скрипта
- `DELETE /scripts/{id}` - Удаление скрипта

### Права доступа
- `GET /projects/{id}/permissions` - Список разрешений
- `POST /projects/{id}/permissions` - Предоставление доступа
- `PUT /projects/{id}/permissions/{user_id}` - Изменение доступа
- `DELETE /projects/{id}/permissions/{user_id}` - Удаление доступа

### FAQ

**Q: Как подключить свою базу данных?**
A: Настройте подключение в бэкенде и обновите конфигурацию API в фронтенде.

**Q: Можно ли использовать Taidon без бэкенда?**
A: Да, включите mock режим в `backendClient.js` для работы с демо данными.

**Q: Как добавить поддержку нового SQL диалекта?**
A: Расширьте конфигурацию CodeMirror в компонентах редактора.

**Q: Поддерживается ли мобильная версия?**
A: Интерфейс адаптивный, но оптимизирован для работы на десктопе.
