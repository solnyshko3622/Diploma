## Введение

Taidon - это веб-приложение для работы с SQL-запросами, предоставляющее удобный интерфейс для написания, валидации и выполнения SQL-кода.

### Основные возможности
- **Редактор SQL** с подсветкой синтаксиса и автодополнением
- **Валидация запросов** в реальном времени
- **Управление проектами** и базами данных
- **История выполненных запросов**
- **Поддержка тем** (светлая/темная)

### Системные требования
- Браузер: Chrome 80+, Firefox 75+, Safari 13+, Edge 80+
- Интернет-соединение для доступа к бэкенду
- Разрешение экрана: 1024x768 и выше

## Быстрый старт

### 1. Регистрация и вход

**Демо-доступ:**
- Email: `demo@example.com`
- Пароль: `password`

**Регистрация нового пользователя:**
1. Нажмите кнопку "Регистрация"
2. Заполните форму:
   - Имя пользователя
   - Email адрес
   - Пароль
3. Нажмите "Зарегистрироваться"

### 2. Создание первого проекта

1. После входа нажмите "Создать проект"
2. Введите название проекта
3. Добавьте описание (опционально)
4. Нажмите "Создать"

### 3. Настройка соединения с БД

1. В проекте нажмите "Добавить соединение"
2. Выберите тип БД (PostgreSQL, MySQL, SQLite)
3. Заполните параметры подключения
4. Нажмите "Подключиться"

## Работа с проектами

### Создание проекта
```
Название: Мой первый проект
Описание: Тестовый проект для работы с БД
Статус: Активный
```

### Управление проектами
- **Просмотр списка**: Главная страница → Проекты
- **Редактирование**: Нажмите на проект → "Редактировать"
- **Удаление**: Нажмите на проект → "Удалить" (только для владельца)

### Члены проекта
- **Добавление участников**: Настройки проекта → Участники
- **Роли**: Владелец, Администратор, Участник, Наблюдатель
- **Права доступа**: Зависят от роли участника

## Управление соединениями

### Поддерживаемые СУБД
- **PostgreSQL** - промышленная реляционная БД
- **MySQL** - популярная open-source БД  
- **SQLite** - встраиваемая легковесная БД

### Создание соединения

**Для PostgreSQL/MySQL:**
```yaml
Название: Продакшен БД
Тип: PostgreSQL
Хост: localhost
Порт: 5432
База данных: mydb
Пользователь: admin
Пароль: ********
SSL: Да/Нет
```

**Для SQLite:**
```yaml
Название: Локальная БД
Тип: SQLite  
Файл: /path/to/database.db
Только чтение: Нет
```

### Тестирование соединения
После заполнения параметров нажмите "Проверить соединение" для проверки доступности БД.

## Редактор SQL

### Интерфейс редактора

```
[Вкладки запросов] [Кнопка выполнения]
───────────────────────────────────────
[Редактор SQL с подсветкой]
───────────────────────────────────────
[Результаты выполнения] [Ошибки]
```

### Основные функции

**Подсветка синтаксиса:**
- Ключевые слова SQL - синий
- Строки - зеленый
- Комментарии - серый
- Числа - оранжевый

**Автодополнение:**
- Начните вводить ключевое слово SQL
- Используйте Ctrl+Space для вызова подсказок
- Выбирайте предложения стрелками и Enter

**Валидация ошибок:**
- Ошибки подсвечиваются красной волнистой линией
- Наведите курсор для просмотра описания ошибки
- Основные типы ошибок:
  - Синтаксические ошибки
  - Неправильная последовательность ключевых слов
  - Незакрытые кавычки и скобки

### Горячие клавиши

| Комбинация | Действие |
|------------|----------|
| `Ctrl+Enter` | Выполнить запрос |
| `Ctrl+S` | Сохранить запрос |
| `Ctrl+Z` | Отменить действие |
| `Ctrl+Y` | Повторить действие |
| `Ctrl+F` | Поиск в запросе |
| `Ctrl+H` | Замена текста |

## Выполнение запросов

### Процесс выполнения

1. **Написание запроса**: Введите SQL код в редактор
2. **Валидация**: Система автоматически проверяет синтаксис
3. **Выполнение**: Нажмите Ctrl+Enter или кнопку "Выполнить"
4. **Результат**: Просмотр результатов в нижней панели

### Примеры запросов

**Простой SELECT:**
```sql
SELECT * FROM users WHERE age > 18;
```

**Создание таблицы:**
```sql
CREATE TABLE products (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    price DECIMAL(10,2)
);
```

**Вставка данных:**
```sql
INSERT INTO products (name, price) 
VALUES ('Тестовый продукт', 99.99);
```

### Обработка результатов

**Успешное выполнение:**
- Отображается таблица с результатами
- Показывается количество строк
- Отображается время выполнения

**Ошибки выполнения:**
- Подробное сообщение об ошибке
- Указание строки с ошибкой
- Рекомендации по исправлению

## История запросов

### Просмотр истории
- Перейдите в раздел "История запросов"
- Фильтрация по проекту и соединению
- Поиск по тексту запроса

### Информация о запросе
Для каждого запроса сохраняется:
- Текст SQL запроса
- Время выполнения
- Количество обработанных строк
- Статус выполнения (успех/ошибка)
- Время выполнения

### Повторное выполнение
Из истории можно:
- Просмотреть детали запроса
- Повторить выполнение
- Скопировать текст запроса

## Настройки системы

### Темы оформления

**Светлая тема:**
- Белый фон редактора
- Черный текст
- Подходит для дневной работы

**Темная тема:**
- Темный фон редактора  
- Светлый текст
- Уменьшает нагрузку на глаза

Переключение: Настройки → Внешний вид → Тема

### Настройки редактора

**Автосохранение:**
- Автоматическое сохранение изменений
- Интервал автосохранения: 5 секунд

**Форматирование:**
- Автоформатирование при вводе
- Выравнивание отступов
- Перенос длинных строк

**Шрифт и размер:**
- Шрифт: Fira Code, Consolas
- Размер шрифта: 14px
- Межстрочный интервал: 1.5

### Настройки безопасности

**Таймаут запросов:**
- Максимальное время выполнения: 30 секунд
- Прерывание долгих запросов

**Ограничения:**
- Максимальный размер запроса: 1MB
- Ограничение на количество строк: 10,000

## Часто задаваемые вопросы

### Вопрос: Как восстановить пароль?
**Ответ:** В текущей версии функция восстановления пароля не реализована. Обратитесь к администратору системы.

### Вопрос: Почему не работает автодополнение?
**Ответ:** Проверьте:
1. Подключение к интернету
2. Настройки редактора (автодополнение включено)
3. Попробуйте использовать Ctrl+Space для принудительного вызова

### Вопрос: Как экспортировать результаты запроса?
**Ответ:** В текущей версии экспорт в CSV доступен через копирование данных из таблицы результатов.

### Вопрос: Поддерживаются ли хранимые процедуры?
**Ответ:** Да, система поддерживает выполнение хранимых процедур для поддерживаемых СУБД.

### Вопрос: Как увеличить лимит времени выполнения?
**Ответ:** В настройках проекта можно увеличить таймаут запросов до 5 минут.
